# Security policy

## Supported versions

| Version | Security fixes |
|---|---|
| Latest stable major | Yes |
| Older majors | Best effort |

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability. Use the repository's **Security → Report a vulnerability** flow to submit a private GitHub Security Advisory.

Include a minimal reproduction, affected versions, expected impact, and any proposed mitigation. Reports will be acknowledged as soon as practical. A coordinated disclosure date will be agreed upon after the issue is reproduced and a fix is available.

Never include production credentials, tokens, personal data, or private route definitions in a report.
